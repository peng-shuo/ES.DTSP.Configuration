# MVP 第一阶段路线图（基于已确认范围）

## 1. 目标

交付可演示的组态工具最小闭环（功能+数据结构）：

- 项目管理与控制组态启动
- 位号创建/导入与有效性检查
- Program 调度属性配置（周期/相位/优先级）
- 控制方案结构化保存与检查结果返回
- SQLite 单文件持久化与单实例窗口复用

---

## 2. 里程碑

## M1：工程骨架与数据底座

**目标**：工程可打开，SQLite 初始化稳定。

**任务**

1. 固化 SQLite 默认连接配置与启动校验。
2. 定义 MVP 数据表（项目、Program、位号、控制方案、功能块实例、连接关系、编辑文档）。
3. 打通 `initDb -> repository -> basic CRUD` 最小链路。

**完成标准**

- 能创建并读取基础工程数据。

---

## M2：主窗口与子窗口机制

**目标**：从主窗口打开编辑器子窗口并实现单实例复用。

**任务**

1. 主进程维护窗口注册表（`program_id + editor_kind -> window_id`）。
2. 重复打开逻辑：已有窗口聚焦，无窗口则创建。
3. 主窗口与子窗口基础 IPC（打开、关闭、保存通知）。

**完成标准**

- 同一 `scheme_id + editor_kind` 重复打开不产生新窗口。

---

## M3：图形程序编辑与方案结构保存

**目标**：支持图形程序组态并可保存为可校验的结构化方案。

**任务**

1. 图形程序基础组态（功能块、位号引用、连接关系）。
2. 方案结构序列化与反序列化。
3. 保存后返回结构化检查结果并可重开恢复。

**完成标准**

- 可完成单回路/串级/分程至少一类场景的结构化保存与恢复。

---

## M4：文本程序编辑与 Program 配置

**目标**：支持文本程序编辑与 Program 调度属性配置。

**任务**

1. 文本程序编辑与保存流程。
2. Program 周期/相位/优先级读写。
3. 重开恢复验证。

**完成标准**

- 文本程序与 Program 调度属性可编辑、保存、重开恢复。

---

## M5：联调与验收

**目标**：完成 MVP 演示脚本。

**任务**

1. 走通端到端演示流程（主窗 -> 位号 -> Program -> 控制方案 -> 保存检查 -> 重开验证）。
2. 修复阻断问题，形成 MVP 发布说明。

**完成标准**

- 满足 `docs/requirements/mvp_phase1_scope.md` 的全部 AC。

---

## 3. 风险与应对

1. **多窗口状态同步复杂度增加**  
   - 应对：MVP 仅做显式保存，减少实时同步负担。
2. **控制方案结构后续扩展风险**  
   - 应对：持久化用领域结构，编辑器载荷与业务模型分层。
3. **范围蔓延**  
   - 应对：严格执行 Out of Scope（无运行层联调、无编译导出）。
