## 2. MVP 范围（In Scope）

### 2.1 功能范围（按用户故事）

1. **项目与工程管理**
   - 在“项目管理”入口启动控制组态；
   - 新建或打开控制项目。
2. **位号管理**
   - 支持手动创建位号（模入/模出/开入/开出等）；
   - 支持位号有效性检查；
   - 支持从外部清单导入位号；
   - 支持从模型工具/流程图工具同步新增位号。
3. **控制方案程序组态**
   - 支持创建用户程序（Program）；
   - 支持配置程序调度属性（周期、起始相位、优先级）；
   - 支持在程序中创建功能块实例、位号引用与连接关系；
   - 支持典型场景：单回路、串级、分程（含分程参数配置）。
4. **保存与检查**
   - 支持显式保存；
   - 支持保存前/后的结构化检查（数据完整性、连接有效性）。
5. **窗口行为**
   - 同一 `Program + editor_kind` 单实例复用（重复打开聚焦已有窗口）。
6. **数据落盘**
   - MVP 使用 SQLite 单文件工程。

### 2.2 已冻结业务策略

1. 位号校验：唯一性 + 必填项 + 命名规则 + 类型匹配。
2. 位号导入：冲突时交互选择（覆盖/跳过/重命名）。
3. 保存检查：允许带警告保存，错误级问题阻断。
4. Program 调度枚举：固定 `fast/1x/2x/5x/10x`。
5. 分程参数：MVP 仅支持 2 段（0~50，50~100）。
6. 窗口复用键：`scheme_id + editor_kind`。

## 3. 非范围（Out of Scope）

以下明确不进入 MVP：

1. 运行层联调（不启动/联调 C++ 运行时）
2. 编译导出能力（不输出运行可加载产物，仅保留检查能力）
3. 自动保存（仅显式保存）
4. SFC/LD/IL 编辑器
5. MySQL 团队模式（归入后续阶段）

## 4. 用户故事与验收标准

### US-001 项目管理入口

**作为** 组态工程师，**我希望** 在项目管理中启动控制组态并打开项目，**以便** 进入控制方案编辑流程。

**AC**

1. 支持新建控制项目或打开已有项目。
2. 进入项目后可访问程序、位号与控制方案编辑入口。

### US-002 位号管理

**作为** 组态工程师，**我希望** 创建或导入位号并完成有效性检查，**以便** 支撑后续控制方案组态。

**AC**

1. 支持手动新增位号（模入/模出/开入/开出）。
2. 支持导入位号清单并落库。
3. 支持位号有效性检查并可提示问题。

### US-003 控制方案组态

**作为** 组态工程师，**我希望** 组态控制方案程序并配置调度属性，**以便** 完成单回路/串级/分程场景搭建。

**AC**

1. 支持创建 Program，并设置周期、起始相位、优先级。
2. 支持创建功能块实例、位号引用和连接关系。
3. 支持分程控制参数配置（如上下限与正反作用）。

### US-004 保存与检查

**作为** 组态工程师，**我希望** 在保存时进行结构化检查，**以便** 尽早发现配置错误。

**AC**

1. 支持显式保存。
2. 保存时返回检查结果（通过/告警/错误）。
3. 关闭并重开后可恢复最后一次保存数据。

### US-005 单实例窗口策略与 SQLite 单文件工程

**作为** 组态工程师，**我希望** 同一对象避免重复编辑冲突，并在本地离线可运行，**以便** 降低使用门槛。

**AC**

1. 重复打开同一 `Program + editor_kind` 时复用已有窗口。
2. 默认 SQLite 连接可完成初始化、保存与读取流程。
3. 不依赖 MySQL 服务即可完成 MVP 演示。

---

## 5. 验收演示脚本（MVP）

1. 启动应用，进入项目管理并新建/打开项目。
2. 创建位号（手工与导入两种方式至少验证一种），完成有效性检查。
3. 创建 Program，配置周期/相位/优先级。
4. 在控制方案中创建功能块实例、位号引用与连接关系。
5. 保存并执行结构化检查，确认结果可见。
6. 关闭并重开项目，验证数据恢复。
7. 重复打开同一 `Program + editor_kind`，验证窗口复用。
